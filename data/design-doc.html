MetricStream - MobileApp

Technical Design Specification

Introduction

Document Objective

This Technical Design Specification (TDS) document elucidates communication protocol of MetricStream MobileApp with MetricStream application. This document will serve as input to AppStudio design team for application design and development. The target audience for this document is AppStudio development and review team.

This document also includes brief description of following components, which are required to establish the communication protocol:

  • MobileApp UI
  • RESTful services
  • Webservices (SOAP message)
  • MetricStream Application

Release Scope

MobileApp will be hosted on iPad and MetricStream Application will be hosted on requisite application server using ECP version 6.0 SP4 and above.

Technical Components

Following are the technical components which would make the MobileApp functional:

  • Mobile App User Interface

    The MobileApp UI will be a lightweight graphical user-interface (GUI) ported on a desired tablet. This UI would be shredded version of the associated form hosted on MetricStream application. The MobileApp UI will provide to the end-user the capability to view an assignment and take requisite action on it. However, user will not have options viz. running an Infolet, scheduling an Infolet, changing user-profile or org-role pair which are intrinsic to application. This UI would not contain any processing capability viz. translating stored-value to corresponding display value or performing JavaScript validation on the UI. From a technical perspective, this App will have the capability to accept and return requisite JSON string to RESTFul web services for communication with concerned MetricStream application server.`

  • RESTful web services layer

    As REST facilitates louse coupling between web-servers, is less strongly typed than SOAP, does not require message header for processing web service request, does not require XML parsing and consumes less bandwidth, it serves as a viable option of communication between MobileApp UI and Platform enabled web services. REST layer will accept json string and invoke corresponding web service. REST services will run as container on an independent server.

  • ECP (Platform) SOAP web services

    The web services stack provided by Platform team would enable one to create and deploy secure, reliable, transactional and interoperable web services for any MetricStream application form. These deployed web service components will transact with RESTFul services for forming request to and response from MetricStream Application server.

  • Web server

    This will be the application server hosting the MetricStream Application. This server would have a web server port exposed for web services.

Technologies used

Front-end

The technologies used on the user interface layer are as follows:

  • Standard web technologies: HTML5, CSS3, JavaScript
  • Application Container
    • Apache Cordova phonegap.com Apache License 2.0

      PhoneGap (previously called Apache Callback, but now Apache Cordova) is an open-source mobile development framework produced by Nitobi, purchased by Adobe Systems. It enables software programmers to build applications for mobile devices using JavaScript, HTML5 and CSS3, instead of device specific languages such as Objective-C. The resulting applications are hybrid, meaning that they are neither truly native (because all layout rendering is done via web views instead of the platform's native UI framework) nor purely web-based (because they are not just web apps, but are packaged as apps for distribution and have access to native device APIs).

  • Libraries
    • KnockoutJS knockoutjs.com MIT LICENSE

      Knockout is a JavaScript library that allows developers to create rich, responsive display and editor user interfaces with a clean underlying data model.
